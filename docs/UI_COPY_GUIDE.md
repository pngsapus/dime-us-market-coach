# UI Copy Guide

## Use

- ควรติดตาม
- รอจังหวะ
- ควรระวัง
- ข้อมูลไม่พอ
- ไม่ควรไล่ซื้อ
- แผนยังอยู่ในเกณฑ์
- แผนวิเคราะห์จำลอง
- ตรวจสอบราคาจริงใน Dime
- ข้อมูลนี้ไม่ใช่ราคาจาก Dime โดยตรง
- ใช้เพื่อประกอบการพิจารณาเท่านั้น ไม่ใช่คำสั่งซื้อ
- อ่านคำอธิบาย
- ดูแผนวิเคราะห์จำลอง
- กลับไป Dashboard

## Do Not Use

- ซื้อเลย
- เข้าทันที
- Strong Buy
- Buy Now
- กำไรแน่
- หุ้นเด็ด
- สัญญาณเข้า

## Tone

Thai copy should be polished, calm, and educational. Avoid language that pressures the user to act.

## Phase 1C UI State Copy

- For mock data, explicitly say `ข้อมูลจำลอง`.
- For Dime price checks, use `ราคาที่เห็นใน Dime` and clarify that users enter this manually.
- For stale or unavailable data, prefer `ข้อมูลล่าช้า`, `ไม่สามารถเชื่อมต่อข้อมูลจำลองจาก backend ได้`, or `ข้อมูลไม่เพียงพอ`.
- For action buttons, use flow language such as `ไปที่ Radar`, `ไปที่แผนวิเคราะห์จำลอง`, `ตรวจสอบราคาจริงใน Dime`, and `บันทึกบทเรียนใน Journal`.
- Keep disclaimers visible near analysis and Dime Check results: `ใช้เพื่อประกอบการพิจารณาเท่านั้น ไม่ใช่คำสั่งซื้อ`.

## Phase 2A.1 Radar Copy

- Treat the sidebar as main navigation.
- In-page buttons should be framed as `ขั้นตอนแนะนำ` or `ไปวิเคราะห์ต่อ`, not a forced wizard.
- Radar title: `Radar หุ้นที่ควรติดตาม`.
- Radar helper copy should explain that users can choose a stock to study or use the sidebar to jump elsewhere.
- Keep one clear mock/local warning near the top; avoid repeating long disclaimers inside every card.
- Discovery traces should be Thai and beginner-facing, for example:
  - `ระบบเริ่มจากรายการหุ้นจำลองในเครื่อง`
  - `คำนวณคะแนนจากแนวโน้ม โมเมนตัม คุณภาพ สภาพคล่อง และความเหมาะสมกับมือใหม่`
  - `หักคะแนนจากความเสี่ยงด้านมูลค่าและความผันผวน`
  - `สรุปสถานะเป็น ... เพื่อช่วยเลือกว่าจะศึกษาต่อหรือควรรอก่อน`
